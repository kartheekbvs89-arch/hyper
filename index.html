What Are Hyperparameters?

Before tuning, you need to deeply understand what you're tuning — and why it matters so much.

Parameters vs Hyperparameters
AspectParametersHyperparameters
Learned by?Model during trainingYou, before training
ExamplesWeights, biaseslearning rate, n_estimators
Updated viaGradient descentSearch algorithms
Stored inmodel.weights_model.get_params()
💡
Key insight: Hyperparameters define the structure and learning process of a model. Getting them wrong means your model will never reach its potential — even with perfect data.
The Hyperparameter Tuning Loop
1
Define search space
Choose which hyperparameters to tune and their possible value ranges
2
Select search strategy
Grid, Random, Bayesian, Evolutionary — each has trade-offs
3
Evaluate with cross-validation
Use k-fold CV to get reliable performance estimates
4
Pick best configuration
Select the config that maximizes your chosen metric
5
Retrain & validate
Retrain on full data with best params, validate on held-out test set
🎛️
Model Complexity Hyperparams
Control how powerful your model is — max_depth, n_estimators, C, gamma. Too high → overfitting. Too low → underfitting.
📐
Regularization Hyperparams
Alpha, lambda, dropout, L1/L2 penalties. These prevent overfitting by penalizing complex models during training.
⚡
Optimization Hyperparams
Learning rate, batch size, momentum, weight decay. These control how the model learns — critical for deep learning.
🏗️
Architecture Hyperparams
Number of layers, hidden units, kernel size, number of heads. Define the structure of neural networks.
🔁
Training Process Hyperparams
Epochs, patience (early stopping), warmup steps, scheduler type. Control the training dynamics over time.

Search Methods

All Tuning Methods Compared

Choose the right method based on your budget, model complexity, and time constraints.

Method How it works Best for Speed Quality Library
Grid Search Exhaustively tries every combination in the grid <3 hyperparams, small search space Very slow Exhaustive GridSearchCV
Random Search Randomly samples from distributions Many hyperparams, limited budget Moderate Good RandomizedSearchCV
Bayesian (GP) Builds a surrogate model of the objective function Expensive evals, continuous params Moderate Excellent skopt
TPE (Optuna) Tree-structured Parzen Estimators model good/bad configs General purpose, mixed spaces Fast Excellent optuna
Hyperband Multi-fidelity: allocate budget to promising configs DL with early stopping possible Very fast Good ray[tune]
CMA-ES Evolution strategy with covariance matrix adaptation Continuous, high-dim search spaces Moderate Excellent optuna / cmaes
Successive Halving Eliminates poor configs early, promotes survivors Large candidate pools, fast evals Very fast Good HalvingGridSearchCV
PBT Population trains in parallel; weak agents copy strong ones Large DL models, RL training Parallel Excellent ray[tune]

🔵 How Bayesian Optimization Works — Intuition

Imagine you're trying to find the tallest mountain in a fog. You can only query one point at a time. Bayesian optimization builds a probabilistic map of the landscape after each query and decides intelligently where to look next.

🗺️
Surrogate Model
Gaussian Process approximates the true objective function
⚡
Acquisition Function
EI balances exploitation (near known best) + exploration (uncertain regions)
🎯
Update & Repeat
Evaluate the point, update the model, pick next point. Gets smarter each step.

Module 2 — Grid Search

grid_search.py
from sklearn.model_selection import GridSearchCV, StratifiedKFold
from sklearn.ensemble import RandomForestClassifier
from sklearn.datasets import load_breast_cancer
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline
import pandas as pd

# Load data
X, y = load_breast_cancer(return_X_y=True)

# Build a pipeline: scaler + model (prevents data leakage!)
pipe = Pipeline([
    ('scaler', StandardScaler()),
    ('rf', RandomForestClassifier(random_state=42, n_jobs=-1))
])

# Define the hyperparameter grid
# NOTE: prefix param name with step name + '__'
param_grid = {
    'rf__n_estimators': [100, 200, 300],
    'rf__max_depth': [None, 5, 10, 20],
    'rf__min_samples_split': [2, 5, 10],
    'rf__max_features': ['sqrt', 'log2', 0.5],
}
# Total combinations: 3 × 4 × 3 × 3 = 108 fits × 5 folds = 540 models

# Stratified k-fold ensures class balance in each fold
cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)

grid_search = GridSearchCV(
    estimator=pipe,
    param_grid=param_grid,
    cv=cv,
    scoring='roc_auc',      # Use AUC for imbalanced data
    n_jobs=-1,              # Use all CPU cores
    verbose=1,
    return_train_score=True  # Detect overfitting
)

grid_search.fit(X, y)

# Results
print(f"Best AUC: {grid_search.best_score_:.4f}")
print(f"Best params: {grid_search.best_params_}")

# All results as DataFrame — great for analysis
results = pd.DataFrame(grid_search.cv_results_)
results_sorted = results.sort_values('rank_test_score')
print(results_sorted[[
    'param_rf__n_estimators', 'param_rf__max_depth',
    'mean_test_score', 'std_test_score',
    'mean_train_score'  # Compare train vs val to detect overfit
]].head(10))

# Get the best fitted model — ready for prediction!
best_model = grid_search.best_estimator_
💡
Pro tip: Always put your scaler inside the Pipeline before running GridSearchCV. If you scale first and then search, you're leaking test fold statistics into training — your results will be overly optimistic!

Module 3 — Random Search with Distributions

random_search.py
from sklearn.model_selection import RandomizedSearchCV
from sklearn.ensemble import GradientBoostingClassifier
from scipy.stats import loguniform, uniform, randint
import numpy as np

# ── Key insight ──────────────────────────────────────────────────────────────
# Use DISTRIBUTIONS, not lists, for continuous hyperparameters.
# loguniform is essential for learning_rate — you want to explore
# 0.001, 0.01, 0.1 equally, not just the endpoints.
# ─────────────────────────────────────────────────────────────────────────────

param_dist = {
    'n_estimators': randint(100, 1000),           # Integer uniform [100, 1000)
    'learning_rate': loguniform(1e-4, 0.5),         # Log scale! 0.0001 to 0.5
    'max_depth': randint(2, 12),
    'subsample': uniform(0.5, 0.5),                # Uniform [0.5, 1.0]
    'min_samples_leaf': randint(1, 20),
    'max_features': uniform(0.3, 0.7),             # [0.3, 1.0]
    'min_samples_split': randint(2, 20),
}

rs = RandomizedSearchCV(
    GradientBoostingClassifier(random_state=42),
    param_distributions=param_dist,
    n_iter=100,           # 100 random samples (vs 1000s for grid)
    cv=5,
    scoring='f1_weighted',
    n_jobs=-1,
    random_state=42,      # Reproducibility
    verbose=2

rs.fit(X_train, y_train)

# Visualize search coverage

Module 6 — Optuna (Bayesian / TPE)

optuna_xgboost.py
import optuna
from optuna.samplers import TPESampler
import xgboost as xgb
from sklearn.model_selection import cross_val_score

# Optuna's "define-by-run" API — the search space IS the function
def objective(trial):
    # ── Suggest hyperparameters for this trial ──────────────────────────────
    params = {
        'n_estimators': trial.suggest_int('n_estimators', 100, 1000, step=50),
        'max_depth': trial.suggest_int('max_depth', 3, 12),
        'learning_rate': trial.suggest_float('learning_rate', 1e-4, 0.3, log=True),
        'subsample': trial.suggest_float('subsample', 0.5, 1.0),
        'colsample_bytree': trial.suggest_float('colsample_bytree', 0.4, 1.0),
        'reg_alpha': trial.suggest_float('reg_alpha', 1e-8, 10.0, log=True),
        'reg_lambda': trial.suggest_float('reg_lambda', 1e-8, 10.0, log=True),
        'gamma': trial.suggest_float('gamma', 0, 5),
        'min_child_weight': trial.suggest_int('min_child_weight', 1, 10),
        'tree_method': 'hist',   # Fast histogram-based training
        'eval_metric': 'logloss',
        'random_state': 42,
    }
    model = xgb.XGBClassifier(**params)
    scores = cross_val_score(model, X, y, cv=5, scoring='roc_auc', n_jobs=-1)
    return scores.mean()   # Optuna maximizes this by default

# Create and run the study
study = optuna.create_study(
    direction='maximize',
    sampler=TPESampler(seed=42),
    study_name='xgb_tuning',
    # storage='sqlite:///optuna_study.db'  # Persist to disk!
)

# Add pruning callback — kills bad trials early (like early stopping)
study.optimize(
    objective,
    n_trials=150,
    timeout=600,          # Stop after 600 seconds
    show_progress_bar=True
)

print(f"\nBest trial: {study.best_trial.number}")
print(f"Best AUC:   {study.best_value:.4f}")
print(f"Best params:")
for key, val in study.best_params.items():
    print(f"  {key}: {val}")

# ── Visualizations ───────────────────────────────────────────────────────────
from optuna.visualization import (
    plot_optimization_history,
    plot_param_importances,
    plot_contour,
    plot_parallel_coordinate
)
plot_optimization_history(study).show()    # Convergence
plot_param_importances(study).show()       # Which param matters most?
plot_contour(study, params=['learning_rate', 'max_depth']).show()
plot_parallel_coordinate(study).show()    # All params, all trials

Module 5 — Deep Learning Hyperparameter Tuning (PyTorch + Optuna)

dl_tuning_pytorch.py
import optuna
import torch
import torch.nn as nn
import torch.optim as optim
from torch.utils.data import DataLoader, TensorDataset

device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')

def build_model(trial, input_dim):
    """Dynamically build network based on trial suggestions."""
    # Tune the number of layers itself!
    n_layers = trial.suggest_int('n_layers', 1, 5)
    dropout = trial.suggest_float('dropout', 0.1, 0.5)
    activation_name = trial.suggest_categorical('activation', ['relu', 'leaky_relu', 'gelu'])
    activations = {'relu': nn.ReLU(), 'leaky_relu': nn.LeakyReLU(0.1), 'gelu': nn.GELU()}

    layers = []
    in_features = input_dim
    for i in range(n_layers):
        out_features = trial.suggest_int(f'n_units_l{i}', 32, 512, log=True)
        layers += [
            nn.Linear(in_features, out_features),
            nn.BatchNorm1d(out_features),
            activations[activation_name],
            nn.Dropout(dropout),
        ]
        in_features = out_features
    layers.append(nn.Linear(in_features, 1))
    return nn.Sequential(*layers)

Module 7 — Successive Halving (Built into sklearn!)

successive_halving.py
from sklearn.experimental import enable_halving_search_cv  # Enable first!
from sklearn.model_selection import HalvingRandomSearchCV
from sklearn.ensemble import RandomForestClassifier
from scipy.stats import randint, uniform

# ── How Successive Halving works ─────────────────────────────────────────────
# Round 1: 512 candidates, each trained on 1/8 of data → keep top 50%
# Round 2: 256 candidates, each trained on 1/4 of data → keep top 50%
# Round 3: 128 candidates, each trained on 1/2 of data → keep top 50%
# Round 4: 64  candidates, each trained on full data   → pick winner!
# Much cheaper than running all 512 on full data.
# ─────────────────────────────────────────────────────────────────────────────

param_dist = {
    'n_estimators': randint(50, 500),
    'max_depth': randint(2, 20),
    'min_samples_split': randint(2, 20),
    'min_samples_leaf': randint(1, 10),
    'max_features': uniform(0.2, 0.8),
}

sh = HalvingRandomSearchCV(
    RandomForestClassifier(n_jobs=-1, random_state=42),
    param_distributions=param_dist,
    n_candidates=512,     # Start with 512 random configs
    factor=2,             # Halve each round (keep top 50%)
    resource='n_samples', # Increase training data each round
    min_resources='exhaust',
    aggressive_elimination=True,
    cv=5,
    scoring='accuracy',
    n_jobs=-1,
    random_state=42
)

sh.fit(X_train, y_train)

# Inspect rounds
results = pd.DataFrame(sh.cv_results_)
print(results[['iter', 'n_resources', 'mean_test_score', 'n_candidates_']])
print(f"\nBest: {sh.best_score_:.4f} with {sh.best_params_}")

Module 10 — Full Pipeline Tuning + Nested CV

nested_cv_pipeline.py
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler, PowerTransformer
from sklearn.decomposition import PCA
from sklearn.feature_selection import SelectFromModel
from sklearn.svm import SVC
from sklearn.model_selection import cross_val_score, StratifiedKFold, RandomizedSearchCV
from scipy.stats import loguniform

# Full pipeline: transformation + feature selection + model
pipe = Pipeline([
    ('scaler', PowerTransformer(method='yeo-johnson')),
    ('feat_sel', SelectFromModel(RandomForestClassifier(n_estimators=50, n_jobs=-1))),
    ('pca', PCA()),
    ('clf', SVC(probability=True))
])

# Tune everything — preprocessing AND model hyperparams together
param_dist = {
    'feat_sel__threshold': ['mean', 'median', '0.5*mean'],
    'feat_sel__estimator__max_depth': randint(2, 8),
    'pca__n_components': uniform(0.7, 0.29),          # % variance to retain
    'clf__C': loguniform(1e-3, 1e3),
    'clf__gamma': loguniform(1e-4, 1e1),
    'clf__kernel': ['rbf', 'poly', 'sigmoid'],
}

# ── NESTED CROSS-VALIDATION ────────────────────────────────────────────────
# Inner CV: hyperparameter selection (avoid overfitting search)
# Outer CV: unbiased performance estimate
# If you just use inner CV score → you're leaking! Nested CV is the fix.
# ──────────────────────────────────────────────────────────────────────────

outer_cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
inner_cv = StratifiedKFold(n_splits=3, shuffle=True, random_state=0)

search = RandomizedSearchCV(
    pipe, param_dist, n_iter=50,
    cv=inner_cv, scoring='roc_auc', n_jobs=-1, random_state=42
)

# Outer loop gives unbiased generalization estimate
nested_scores = cross_val_score(search, X, y, cv=outer_cv, scoring='roc_auc', n_jobs=-1)
print(f"Nested CV AUC: {nested_scores.mean():.4f} ± {nested_scores.std():.4f}")
# This is your HONEST model performance estimate!

# Final model: train on all data with best params
search.fit(X, y)
final_model = search.best_estimator_
print(f"Final best params: {search.best_params_}")
⚠️
Critical pitfall: If you tune on your test set (or use the inner CV score as your final estimate), you've overfit your hyperparameters. Nested CV gives you an unbiased estimate of how well your tuning will generalize to truly new data.

Module 8 — Learning Rate Finder + Cosine Warmup

lr_scheduling.py
import torch, math
import torch.optim as optim

# ── Learning Rate Range Test (LR Finder) ──────────────────────────────────
# Start with tiny LR, exponentially increase, plot loss.
# The LR just before loss starts rising is your best LR.
# Rule of thumb: use LR = (peak / 10) for training.
# ──────────────────────────────────────────────────────────────────────────
def lr_range_test(model, train_loader, criterion, start_lr=1e-7, end_lr=10, num_iter=100):
    optimizer = optim.SGD(model.parameters(), lr=start_lr)
    lr_mult = (end_lr / start_lr) ** (1 / num_iter)
    lrs, losses = [], []
    best_loss = float('inf')

    for i, (Xb, yb) in enumerate(train_loader):
        if i >= num_iter: break
        loss = criterion(model(Xb), yb)
        optimizer.zero_grad(); loss.backward(); optimizer.step()
        lrs.append(optimizer.param_groups[0]['lr'])
        losses.append(loss.item())
        best_loss = min(best_loss, loss.item())
        if loss.item() > 4 * best_loss: break   # Stop if loss explodes
        for g in optimizer.param_groups: g['lr'] *= lr_mult
    return lrs, losses

# ── Linear Warmup + Cosine Decay (used in Transformers) ───────────────────
def get_cosine_schedule_with_warmup(optimizer, num_warmup_steps, num_training_steps, min_lr=0):
    def lr_lambda(step):
        if step < num_warmup_steps:
            return step / max(1, num_warmup_steps)  # Linear warmup
        progress = (step - num_warmup_steps) / max(1, num_training_steps - num_warmup_steps)
        return max(min_lr, 0.5 * (1 + math.cos(math.pi * progress)))  # Cosine decay
    return optim.lr_scheduler.LambdaLR(optimizer, lr_lambda)

# Usage: warmup for 10% of steps, cosine decay for the rest
optimizer = optim.AdamW(model.parameters(), lr=3e-4, weight_decay=0.01)
scheduler = get_cosine_schedule_with_warmup(
    optimizer,
    num_warmup_steps=int(0.1 * total_steps),
    num_training_steps=total_steps
)

# In training loop:
for step, (Xb, yb) in enumerate(train_loader):
    loss = criterion(model(Xb), yb)
    optimizer.zero_grad(); loss.backward()
    nn.utils.clip_grad_norm_(model.parameters(), 1.0)
    optimizer.step()
    scheduler.step()   # Call after optimizer.step()!
Advanced Topics

Algorithm Deep Dives

Go beyond the sklearn API and understand what's happening under the hood.

🎰
Random Forest
10–30 hyperparams · High impact
The most important hyperparams are n_estimators, max_features, and max_depth. More trees rarely hurts (just slower). Start with max_features='sqrt' for classification, 'auto' for regression. min_samples_leaf=4 often improves stability on noisy data.
n_estimators: 100–1000 max_depth: None–20 max_features: sqrt, 0.3–0.8 min_samples_leaf: 1–10 bootstrap: True/False
🚀
XGBoost / LightGBM
15+ hyperparams · Critical tuning
The key triad is learning_rate × n_estimators × max_depth. Lower LR requires more trees. Use early stopping instead of tuning n_estimators. For XGB, colsample_bytree and subsample control stochasticity and prevent overfitting. LightGBM's num_leaves replaces max_depth for finer control.
learning_rate: 0.01–0.3 max_depth: 3–10 gamma / min_gain: 0–5 reg_alpha/lambda: log-scale colsample_bytree: 0.4–1.0
🧠
Neural Networks (DL)
20+ hyperparams · Architecture NAS
Learning rate dominates everything — use a LR finder first. Batch size affects gradient noise: smaller = more noise = better generalization but slower. For depth vs width: deeper networks learn more abstract features; wider layers expand capacity. Always use batch norm for deep nets. Weight decay (1e-4 to 1e-2) is often more useful than dropout for structured data.
lr: 1e-5–1e-1 (log) batch_size: 32–512 n_layers: 1–8 hidden_dim: 32–2048 (log) dropout: 0.1–0.5 weight_decay: 1e-5–1e-2
📐
SVM
3–4 key hyperparams · Geometric
C controls the margin hardness vs misclassification trade-off. gamma defines the influence radius of each training point — high gamma = tight fit = overfit. Always search both on log scale. The C-gamma grid is the classic SVM search space. Scale your features first — SVM is not scale-invariant!
C: 1e-3–1e3 (log) gamma: 1e-4–10 (log) kernel: rbf, poly, linear degree: 2–5 (poly only)
⚡
Transformers (BERT/GPT)
Fine-tuning hyperparams · Critical
For fine-tuning: LR is the #1 factor — try 1e-5 to 5e-5. Use linear warmup (10% of steps) + linear or cosine decay. Batch size × gradient accumulation = effective batch size (32–256 typical). Too many epochs → catastrophic forgetting of pretraining. Usually 2–5 epochs is enough. Layer-wise LR decay (LLRD) helps: lower layers get smaller LR.
lr: 1e-5–5e-5 warmup_ratio: 0.06–0.1 weight_decay: 0.01 epochs: 2–5 LLRD: 0.8–0.95 per layer
🔵
Bayesian Opt (Gaussian Process)
Surrogate model tuning
The GP surrogate itself has hyperparams: the kernel (Matérn 5/2 is default in skopt), length scales, and noise level. The acquisition function κ (UCB) or ξ (EI) controls exploration. More exploration early → exploit later. For high-dim spaces (>20 params), GP doesn't scale well — use TPE or CMA-ES instead.
n_initial_points: 10–20 acq_func: EI, UCB, PI kappa (UCB): 1.96 xi (EI/PI): 0.01 kernel: Matern(nu=2.5)
Key Formulas

Important Mathematical Relationships

Expected Improvement acquisition function:

EI(x) = (μ(x) − f* − ξ) · Φ(Z) + σ(x) · φ(Z)
where Z = (μ(x) − f* − ξ) / σ(x)

Cosine annealing LR schedule:

η_t = η_min + ½(η_max − η_min)(1 + cos(πt/T_max))

Hyperband bracket schedule:

s_max = ⌊log_η(B/R)⌋
n_i = ⌈(n · η^i) / (s+1-i)⌉, r_i = R · η^(i-s)

L1 + L2 Elastic Net regularization:

Loss = MSE + α·ρ·||w||₁ + α·(1-ρ)/2·||w||₂²
Best Practices

The Rules Every ML Engineer Knows

Always use log-scale for learning rate, C, gamma, alpha
Scale features before SVM, KNN, logistic regression
Use CV inside the pipeline, never before it
Tune n_estimators last (more is usually better)
For DL: fix LR first, then tune architecture
Use Optuna's pruning — it cuts tuning time by 10×
Nested CV for unbiased performance estimates
Check train vs val score to detect overfitting
Use n_jobs=-1 everywhere you can
Set random_state for reproducibility
Plot optimization history to know when to stop
fANOVA to find which hyperparams actually matter
⚠️
The hardest lesson: Hyperparameter tuning is NOT a substitute for good feature engineering, data cleaning, or picking the right model family. Get your baseline solid first, then tune. Squeezing 0.001 AUC with 10 hours of Bayesian search when your data has errors is wasted effort.

Quick Reference

When to Use Which Method

🔢
Use Grid Search when...
You have ≤3 hyperparameters, a small search space, need exhaustive coverage, or are verifying a specific hypothesis about a range of values. Fast with small grids.
🎲
Use Random Search when...
You have 5+ hyperparameters, a limited evaluation budget, or continuous search spaces. Set n_iter = 50–200 with scipy distributions. Often 60× faster than grid.
🔵
Use Bayesian Opt when...
Each model evaluation is expensive (≥30s), you have a moderate search space (3–15 params), and need the best possible result within a fixed trial budget of 50–200 runs.
⚡
Use Hyperband/SH when...
You're training deep learning models where early stopping is meaningful. You have many candidate configs and want to eliminate losers early. Use with Ray Tune or Optuna.
🤖
Use AutoML when...
You need a strong baseline fast, you're exploring multiple algorithm families at once, or you're in early-stage experimentation. Auto-sklearn, FLAML, or H2O handle model selection + tuning together.
