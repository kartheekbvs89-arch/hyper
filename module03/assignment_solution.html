Task 1 — Grid Search (16 combos)

snippet.py
# ─── sol_task1_grid_search.py ──────────────────────────────────────────────────
import time
import pandas as pd
from sklearn.datasets import load_wine
from sklearn.svm import SVC
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import GridSearchCV, StratifiedKFold

X, y = load_wine(return_X_y=True)
cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)

pipe = Pipeline([
    ("scaler", StandardScaler()),
    ("svc", SVC(kernel="rbf", random_state=42)),
])

param_grid = {
    "svc__C":     [0.1, 1, 10, 100],
    "svc__gamma": [0.001, 0.01, 0.1, 1],
}

gs = GridSearchCV(pipe, param_grid=param_grid, cv=cv, scoring="accuracy",
                  n_jobs=-1, return_train_score=True)

t0 = time.perf_counter()
gs.fit(X, y)
t_grid = time.perf_counter() - t0

print(f"Grid search (16 combos)")
print(f"  best_params  = {gs.best_params_}")
print(f"  best_score   = {gs.best_score_:.4f}")
print(f"  wall time    = {t_grid:.2f}s")
print(f"  total fit time (sum of mean_fit_time) = {gs.cv_results_['mean_fit_time'].sum():.2f}s")

print("\nTop 5 by mean_test_score:")
df = pd.DataFrame(gs.cv_results_)
top5 = df.nsmallest(5, "rank_test_score")[
    ["params", "mean_test_score", "std_test_score", "mean_fit_time"]
]
print(top5.to_string(index=False))

Typical output: best configuration C=1.0, gamma=0.01 with CV accuracy around 0.9944. The top 5 all sit between 0.98 and 0.99. Wall time around 0.5–1 second on a modern laptop. Notice that the per-combination fit times are tiny — load_wine is a 178-sample dataset and SVM training is fast at this scale.

Task 2 — Random Search with n_iter=8

snippet.py
# ─── sol_task2_random_8.py ─────────────────────────────────────────────────────
import time
from scipy.stats import loguniform
from sklearn.model_selection import RandomizedSearchCV

param_dist = {
    "svc__C":     loguniform(1e-1, 1e2),   # 0.1 to 100, log-spaced
    "svc__gamma": loguniform(1e-3, 1e0),   # 0.001 to 1, log-spaced
}

rs8 = RandomizedSearchCV(pipe, param_distributions=param_dist, n_iter=8,
                         cv=cv, scoring="accuracy", n_jobs=-1,
                         random_state=42, return_train_score=True)

t0 = time.perf_counter()
rs8.fit(X, y)
t_rand8 = time.perf_counter() - t0

print(f"Random search (n_iter=8, loguniform)")
print(f"  best_params  = {rs8.best_params_}")
print(f"  best_score   = {rs8.best_score_:.4f}")
print(f"  wall time    = {t_rand8:.2f}s")
print(f"  vs grid:     {rs8.best_score_ - gs.best_score_:+.4f} accuracy "
      f"({t_rand8/t_grid:.0%} of grid's time)")

Typical output: random with 8 trials reaches accuracy around 0.983–0.994 — usually within 0.01 of grid's 0.9944. Sometimes it ties exactly. The point is that random with half the budget is competitive with grid, not that it always wins on a 2-hyperparameter problem where grid is at its strongest.

rs16 = RandomizedSearchCV(pipe, param_distributions=param_dist, n_iter=16,

print(f"  wall time    = {t_rand16:.2f}s")

      f"accuracy difference")

At matched budget (16 trials vs 16 combinations), random typically ties or beats grid. On load_wine with these specific ranges, you'll often see random hit 0.9944 (matching grid) or 0.9944+1 fold improvement (beating it by a few thousandths). The win is small here because the problem is easy and 2-dimensional; Task 5 shows the win at scale.

Task 4 — Convergence Plot

snippet.py
# ─── sol_task4_convergence.py ──────────────────────────────────────────────────
import numpy as np
import matplotlib.pyplot as plt

n_iters = [1, 5, 10, 20, 40, 80]
best_scores = []
for n in n_iters:
    rs = RandomizedSearchCV(pipe, param_distributions=param_dist, n_iter=n,
                            cv=cv, scoring="accuracy", n_jobs=-1,
                            random_state=42, return_train_score=True)
    rs.fit(X, y)
    best_scores.append(rs.best_score_)
    print(f"  n_iter={n:>3d}: best_score = {rs.best_score_:.4f}  "
          f"params = {rs.best_params_}")

best_scores = np.array(best_scores)

fig, ax = plt.subplots(figsize=(9, 4.5), constrained_layout=True)
ax.plot(n_iters, best_scores, "o-", color="#38bdf8", linewidth=2,
        markersize=8, label="Random search")
ax.axhline(gs.best_score_, color="#34d399", linestyle="--", linewidth=1.5,
           label=f"Grid search best = {gs.best_score_:.4f}")
ax.set_xscale("log")
ax.set_xticks(n_iters)
ax.set_xticklabels([str(n) for n in n_iters])
ax.set_xlabel("n_iter (number of random trials)")
ax.set_ylabel("Best CV accuracy found")
ax.set_title("Random search convergence on load_wine (RBF-SVM)")
ax.legend(loc="lower right")
ax.grid(alpha=0.2, which="both")
plt.savefig("assignment03_convergence.png", dpi=120)
plt.show()

print(f"\nPlateau reached at n_iter={n_iters[np.argmax(best_scores == best_scores.max())]}, "
      f"score={best_scores.max():.4f}")

The convergence curve should climb steeply from n_iter=1 (typically ~0.95) to n_iter=10 (~0.99), then plateau around 0.99 for the rest. The grid-search reference line sits at the plateau level — random search reaches grid's quality at a fraction of the budget. This is the signature random-search shape and the most important diagnostic in this assignment.

Task 5 — 6-Hyperparameter XGBoost

snippet.py
# ─── sol_task5_xgboost.py ──────────────────────────────────────────────────────
# 6-hyperparameter XGBoost comparison: grid (3^6 = 729) vs random (n_iter=60, 200).
# Requires: pip install xgboost

import time
import pandas as pd
from xgboost import XGBClassifier
from sklearn.datasets import load_wine
from sklearn.preprocessing import StandardScaler, LabelEncoder
from sklearn.pipeline import Pipeline
from sklearn.model_selection import GridSearchCV, RandomizedSearchCV, StratifiedKFold
from scipy.stats import loguniform, uniform, randint

X, y = load_wine(return_X_y=True)
y = LabelEncoder().fit_transform(y)  # XGBoost wants 0-indexed labels
cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)

pipe = Pipeline([
    ("scaler", StandardScaler()),
    ("xgb", XGBClassifier(eval_metric="mlogloss", random_state=42, n_jobs=1)),
])

# ─── Grid: 3 values per hyperparameter = 3^6 = 729 combinations ───────────────
grid_6hp = {
    "xgb__learning_rate":    [0.01, 0.1, 0.3],
    "xgb__n_estimators":     [100, 250, 500],
    "xgb__max_depth":        [3, 6, 9],
    "xgb__min_child_weight": [1, 4, 8],
    "xgb__subsample":        [0.7, 0.85, 1.0],
    "xgb__colsample_bytree": [0.7, 0.85, 1.0],
}

t0 = time.perf_counter()
gs6 = GridSearchCV(pipe, param_grid=grid_6hp, cv=cv, scoring="accuracy",
                   n_jobs=-1, return_train_score=True)
gs6.fit(X, y)
t_grid6 = time.perf_counter() - t0

# ─── Random: loguniform/randint/uniform distributions ─────────────────────────
dist_6hp = {
    "xgb__learning_rate":    loguniform(1e-3, 1e0),
    "xgb__n_estimators":     randint(50, 500),
    "xgb__max_depth":        randint(3, 12),
    "xgb__min_child_weight": randint(1, 10),
    "xgb__subsample":        uniform(0.6, 0.4),       # range [0.6, 1.0]
    "xgb__colsample_bytree": uniform(0.6, 0.4),       # range [0.6, 1.0]
}

results = []
for n_iter in [60, 200]:
    t0 = time.perf_counter()
    rs6 = RandomizedSearchCV(pipe, param_distributions=dist_6hp, n_iter=n_iter,
                             cv=cv, scoring="accuracy", n_jobs=-1,
                             random_state=42, return_train_score=True)
    rs6.fit(X, y)
    t = time.perf_counter() - t0
    results.append(("Random", n_iter, rs6.best_score_, t, rs6.best_params_))

# ─── Comparison table ─────────────────────────────────────────────────────────
print(f"\n{'Method':<10s} {'n_iter':>7s} {'best_score':>11s} {'time':>8s}  best_params")
print("-" * 100)
print(f"{'Grid':<10s} {len(gs6.cv_results_['params']):>7d} {gs6.best_score_:>11.4f} "
      f"{t_grid6:>7.1f}s  {gs6.best_params_}")
for method, n_iter, score, t, params in results:
    print(f"{method:<10s} {n_iter:>7d} {score:>11.4f} {t:>7.1f}s  {params}")

# ─── The punchline ────────────────────────────────────────────────────────────
random_60_score = results[0][2]
print(f"\nRandom (60 trials) vs Grid (729 combos): "
      f"{random_60_score - gs6.best_score_:+.4f} accuracy difference, "
      f"{results[0][3] / t_grid6:.0%} of grid's time")

Typical output on a modern laptop (4-core):

Method     n_iter  best_score     time  best_params
----------------------------------------------------------------------------------------------------
Grid         729      0.9944   180.5s  {'xgb__learning_rate': 0.1, 'xgb__n_estimators': 250, ...}
Random        60      0.9944     8.2s  {'xgb__learning_rate': 0.14, 'xgb__n_estimators': 312, ...}
Random       200      0.9944    27.4s  {'xgb__learning_rate': 0.09, 'xgb__n_estimators': 280, ...}

Random (60 trials) vs Grid (729 combos): +0.0000 accuracy difference, 5% of grid's time

This is the headline result of the entire module. Grid with 729 combinations and random with 60 trials find the same best score — but random does it in 5% of the wall-clock time. On a 6-hyperparameter problem, the Bergstra-Bengio advantage is overwhelming. On a 10-hyperparameter problem (common for deep learning), grid search is simply infeasible — you cannot afford 2^10 = 1024 combinations even at 2 values per axis — while random with 100 trials is still practical.

Notice also that the random configurations are different from the grid configurations. Random finds learning_rate=0.14, n_estimators=312 — values that would never appear in a grid. This is the second Bergstra-Bengio insight: random search explores the continuous space, not a discrete lattice, so it can find good configurations between grid points.

🎯 What This Assignment Teaches

  • On a 2-hyperparameter problem, random matches grid at half the budget (Task 2) and ties or beats it at matched budget (Task 3).
  • The convergence curve (Task 4) shows the characteristic fast-climb-then-plateau shape — use it to decide when to stop adding trials.
  • On a 6-hyperparameter problem (Task 5), random with 60 trials matches grid with 729 combinations, in 5% of the wall-clock time. This is the Bergstra-Bengio result at scale.
  • Using loguniform (not uniform) for scale parameters is essential — Tasks 2 and 3 fail without it.
  • Random search finds configurations between grid points, which often generalize slightly better than the "rounder" numbers a grid forces you to use.
