Assignment Overview

In this assignment you will empirically reproduce the central claim of Module 03 — that random search matches or beats grid search at a fraction of the cost — by tuning an RBF-kernel SVM on the load_wine dataset. You will run grid search with 16 combinations, then random search with the same budget (16 trials) and half the budget (8 trials), and confirm that random matches grid at half the cost. You will then plot a convergence curve to see random search's characteristic fast-initial-climb shape, and finally extend the experiment to a 6-hyperparameter XGBoost problem to see how random's advantage grows with dimensionality.

The assignment is designed to take 2–3 hours if you are comfortable with scikit-learn, scipy.stats, and matplotlib. Tasks 1–3 should take 30–45 minutes each; Task 4 (the convergence plot) about 30 minutes; Task 5 (the XGBoost bonus) about an hour if you choose to attempt it. If you find yourself spending more than 4 hours total, re-read the article sections on loguniform and n_iter — most stuck points come from using the wrong distribution or not understanding the convergence curve.

Dataset

We use sklearn.datasets.load_wine — the classic wine cultivar dataset, 178 samples, 13 chemical-feature columns (alcohol, malic acid, ash, alkalinity of ash, magnesium, total phenols, flavanoids, nonflavanoid phenols, proanthocyanins, color intensity, hue, OD280/OD315 of diluted wines, proline), 3 classes (cultivars 0, 1, 2). It is small enough that every experiment runs in seconds, rich enough that hyperparameter choice moves accuracy by 5–10 percentage points, and clean enough that you can focus on the tuning rather than data cleaning.

Hint: where to find the timing

After fitting, gs.cv_results_ is a dict. 'mean_fit_time' and 'std_fit_time' are per-combination timing arrays. 'mean_test_score' and 'std_test_score' are per-combination CV scores. Sum 'mean_fit_time' across all combinations for total fit time (this is the sequential time; with n_jobs=-1 the wall-clock time will be lower).

Task 4 — Convergence Plot

Plot the best-score-found-so-far versus the number of trials, for n_iter in [1, 5, 10, 20, 40, 80]. For each n_iter, run a fresh RandomizedSearchCV with that budget (use the same random_state=42 for fairness), record the best score, and plot. The curve should show a fast initial climb followed by a plateau — the characteristic random-search convergence shape.

Hint: how to make the curve smooth

For each n_iter value, the best score is just search.best_score_. To make the curve smoother and reduce noise from a single random seed, you can repeat each n_iter with 5 different random states (e.g. 42, 43, 44, 45, 46) and plot the mean ± std. But for a first pass, a single seed is fine — the shape will be clear.

Task 5 (Bonus) — 6-Hyperparameter XGBoost Problem

Repeat the grid-vs-random comparison on a 6-hyperparameter XGBoost problem to see how random's advantage grows with dimensionality. Use load_wine (or load_breast_cancer if you prefer a binary problem). Tune the following six hyperparameters:

  • learning_rate: loguniform on [1e-3, 1e0]
  • n_estimators: randint on [50, 500]
  • max_depth: randint on [3, 12]
  • min_child_weight: randint on [1, 10]
  • subsample: uniform on [0.6, 0.4] (i.e. range [0.6, 1.0])
  • colsample_bytree: uniform on [0.6, 0.4] (i.e. range [0.6, 1.0])

For grid search, pick 3 values per hyperparameter (so 3^6 = 729 combinations — expect this to take a few minutes). For random search, use n_iter=60 and n_iter=200. Compare best scores and run times. You should see random with 60 trials matching or beating grid with 729 combinations — that's the Bergstra-Bengio advantage at scale.

💡
Install XGBoost first

If you don't have XGBoost installed, run pip install xgboost in your environment. If you can't install it, swap to HistGradientBoostingClassifier from scikit-learn — its hyperparameters are similar (learning_rate, max_iter, max_leaf_nodes, min_samples_leaf, l2_regularization) and you'll still see the same dimensionality advantage for random search.
