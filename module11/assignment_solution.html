Task 1 — Run the Optuna Study

snippet.py
# ─── sol_task1_run_study.py ────────────────────────────────────────────────────
import optuna
import numpy as np
from xgboost import XGBClassifier
from sklearn.datasets import load_breast_cancer
from sklearn.model_selection import StratifiedKFold, cross_validate

X, y = load_breast_cancer(return_X_y=True)
cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)


def objective(trial):
    params = {
        "learning_rate":     trial.suggest_float("learning_rate", 1e-3, 1.0, log=True),
        "max_depth":         trial.suggest_int("max_depth", 3, 12),
        "n_estimators":      trial.suggest_int("n_estimators", 50, 500),
        "subsample":         trial.suggest_float("subsample", 0.5, 1.0),
        "colsample_bytree":  trial.suggest_float("colsample_bytree", 0.5, 1.0),
        "min_child_weight":  trial.suggest_int("min_child_weight", 1, 10),
        "n_jobs":            -1,
        "eval_metric":       "logloss",
        "random_state":      42,
        "tree_method":       "hist",
    }
    model = XGBClassifier(**params)
    result = cross_validate(
        model, X, y, cv=cv, scoring="accuracy",
        return_train_score=True, n_jobs=-1,
    )
    val_score = float(result["test_score"].mean())
    train_score = float(result["train_score"].mean())
    trial.set_user_attr("train_score", train_score)
    return val_score


# Create or load the study
study = optuna.create_study(
    direction="maximize",
    sampler=optuna.samplers.TPESampler(seed=42),
    storage="sqlite:///assign11.db",
    study_name="assign11_xgb",
    load_if_exists=True,
)
study.optimize(objective, n_trials=100, show_progress_bar=False)

print(f"Best value:      {study.best_value:.4f}")
print(f"Best trial:      #{study.best_trial.number}")
print(f"Best parameters: {study.best_params}")
print(f"Train score at best trial: {study.best_trial.user_attrs['train_score']:.4f}")

Typical output: best validation accuracy ≈ 0.978–0.983. Best parameters typically look like learning_rate ≈ 0.05, max_depth ≈ 5, n_estimators ≈ 300, subsample ≈ 0.85, colsample_bytree ≈ 0.75, min_child_weight ≈ 2. Train score at the best trial is typically 1.000 — a 2–3% train-val gap, well within the "not overfit to validation" range.

Task 2 — Generate the Five Visualizations

snippet.py
# ─── sol_task2_visualizations.py ───────────────────────────────────────────────
import optuna
from optuna.importance import get_param_importances
from pathlib import Path

OUT = Path("assign11_plots"); OUT.mkdir(exist_ok=True)
study = optuna.load_study(study_name="assign11_xgb", storage="sqlite:///assign11.db")

# 1. Optimization history (convergence)
fig1 = optuna.visualization.plot_optimization_history(study)
fig1.write_html(OUT / "01_optimization_history.html")

# 2. Parallel coordinates
fig2 = optuna.visualization.plot_parallel_coordinate(study)
fig2.write_html(OUT / "02_parallel_coords.html")

# 3. Parameter importances
fig3 = optuna.visualization.plot_param_importances(study)
fig3.write_html(OUT / "03_param_importances.html")

# 4. Contour plot for the 2 most important params
importances = get_param_importances(study)
top2 = list(importances.keys())[:2]
print(f"Top-2 most important params: {top2}")
print("Full importance ranking:")
for name, imp in importances.items():
    print(f"  {name:25s} {imp:.3f}")

fig4 = optuna.visualization.plot_contour(study, params=top2)
fig4.write_html(OUT / "04_contour_top2.html")

# 5. Slice plots
fig5 = optuna.visualization.plot_slice(study)
fig5.write_html(OUT / "05_slice_all.html")

print(f"\nAll 5 plots saved to {OUT.resolve()}")

Expected importance ranking (will vary slightly with seed, but the pattern is robust): learning_rate (40–55%), max_depth (20–30%), n_estimators (5–15%), then subsample, colsample_bytree, min_child_weight all under 10%. The contour plot of learning_rate × max_depth should show a clear diagonal ridge of good scores in the (low-LR, mid-depth) region.

Task 3 — Sample 1-Page Analysis

snippet.py
# ─── sol_task3_analysis.md ─────────────────────────────────────────────────────
# Sample 1-page analysis (~500 words). Your analysis will differ in details
# but should follow the same structure: 4 sub-sections, each citing evidence.

# ## Which hyperparameters matter most?
#
# The fANOVA importance ranking for this 100-trial TPE study on XGBoost +
# breast_cancer shows:
#   - learning_rate:        0.512  (51.2% of variance explained)
#   - max_depth:            0.238  (23.8%)
#   - n_estimators:         0.108
#   - subsample:            0.062
#   - colsample_bytree:     0.041
#   - min_child_weight:     0.018
#
# `learning_rate` and `max_depth` together explain ~75% of the variance in
# validation accuracy, which matches the theoretical expectations from
# Module 5 (gradient boosting) and Module 6 (tree ensembles). The remaining
# 4 hyperparameters split ~25%, with `n_estimators` being the most impactful
# of the secondary group.
#
# ## What regions of the hyperparameter space are good?
#
# From the parallel-coordinates plot (filtered to top-20% trials), the best
# trials cluster in:
#   - learning_rate ∈ [0.02, 0.15]  (low end of the search range)
#   - max_depth ∈ [4, 7]
#   - n_estimators ∈ [200, 500]
#   - subsample ∈ [0.75, 0.95]
#   - colsample_bytree ∈ [0.6, 0.9]
#   - min_child_weight ∈ [1, 4]
#
# The good region is consistent with the XGBoost "triad" intuition: low LR
# + many trees + moderate depth. The top-20% filter is essential — on the
# full parallel-coordinates plot, the good region is invisible behind the
# noise of the worse trials.
#
# ## Are there interactions?
#
# The contour plot of `learning_rate × max_depth` shows a clear diagonal
# ridge running from (low LR, low depth) to (mid LR, mid depth). The best
# validation scores live along this ridge; off the ridge, scores drop
# sharply. This is a textbook interaction signature: the optimal
# `max_depth` depends on the `learning_rate` — high LR requires shallow
# trees (the model fits fast and overfits if given too much capacity), low
# LR can afford deeper trees.
#
# The contour is not perfectly diagonal (it has a slight vertical
# component in the high-LR region), which means the interaction is strong
# but not the only effect. fANOVA's `V_ij` term would quantify the
# interaction strength directly, but the visual is clear enough.
#
# ## Has the search converged?
#
# From the optimization history, the best-so-far curve rises sharply in
# the first 12 trials (from ~0.92 to ~0.975), then climbs slowly to ~0.978
# by trial ~40, then is flat for the remaining 60 trials. The last
# improvement was at trial 43; the search ran 57 more trials without
# improvement.
#
# Verdict: converged. Running another 100 trials is unlikely to beat the
# incumbent by more than ~0.1% (within Monte Carlo noise from the 5-fold
# CV). I would stop here and either (a) accept the current best parameters,
# or (b) switch to a finer-grained local search (CMA-ES) initialized at the
# current best.

Task 4 — Overfitting Check

snippet.py
# ─── sol_task4_overfitting_check.py ─────────────────────────────────────────────
import numpy as np
import matplotlib.pyplot as plt
import optuna

study = optuna.load_study(study_name="assign11_xgb", storage="sqlite:///assign11.db")
trials = [t for t in study.trials if t.state == optuna.trial.TrialState.COMPLETE]

val_scores = np.array([t.value for t in trials])
train_scores = np.array([t.user_attrs["train_score"] for t in trials])
gaps = train_scores - val_scores

# Scatter: train (x) vs val (y), colored by val score
fig, ax = plt.subplots(figsize=(7.5, 6), constrained_layout=True)
sc = ax.scatter(train_scores, val_scores, c=val_scores, cmap="viridis",
                s=50, edgecolors="white", linewidth=0.6)
ax.plot([0.90, 1.005], [0.90, 1.005], "--", color="#8899aa",
        label="train = val (no gap)")
fig.colorbar(sc, label="Validation accuracy")
ax.set_xlabel("Mean train accuracy (5-fold)")
ax.set_ylabel("Mean validation accuracy (5-fold)")
ax.set_title("Train vs validation: overfitting check")
ax.legend(loc="lower right")
ax.set_xlim(0.95, 1.005)
ax.set_ylim(0.92, 1.00)
plt.savefig("assign11_task4_train_vs_val.png", dpi=120)
plt.show()

# Quantitative overfit check
top10_idx = np.argsort(val_scores)[-10:]
top10_gap = float(gaps[top10_idx].mean())
overall_gap = float(gaps.mean())
ratio = top10_gap / overall_gap

print(f"Top-10 trials by val score:")
print(f"  mean train score: {train_scores[top10_idx].mean():.4f}")
print(f"  mean val score:   {val_scores[top10_idx].mean():.4f}")
print(f"  mean gap:         {top10_gap:.4f}")
print(f"\nAll trials:")
print(f"  mean gap:         {overall_gap:.4f}")
print(f"\nRatio (top10 / overall): {ratio:.2f}")
if top10_gap > 1.5 * overall_gap:
    print("⚠️  Overfitting signal: top-val trials have a larger train-val gap.")
else:
    print("✓  No overfitting signal in top-val trials.")

Typical output: top-10 mean gap ≈ 0.022, overall mean gap ≈ 0.018, ratio ≈ 1.2. Since 1.2 < 1.5, the check concludes "no overfitting signal." This is the expected result for XGBoost on breast_cancer — the dataset is small but well-behaved, and the implicit regularization (low LR + shallow trees) prevents the search from finding trials that exploit noise in the validation folds.

Task 5 — fANOVA Cross-Evaluator Comparison

snippet.py
# ─── sol_task5_fanova_evaluators.py ─────────────────────────────────────────────
import optuna
from optuna.importance import (
    FanovaImportanceEvaluator,
    MeanDecreaseImpurityImportanceEvaluator,
    get_param_importances,
)
from scipy.stats import spearmanr

study = optuna.load_study(study_name="assign11_xgb", storage="sqlite:///assign11.db")

imps_fanova = get_param_importances(
    study, evaluator=FanovaImportanceEvaluator(seed=42)
)
imps_mdi = get_param_importances(
    study, evaluator=MeanDecreaseImpurityImportanceEvaluator(seed=42)
)

print(f"{'Hyperparameter':25s}  {'fANOVA':>8s}  {'MDI':>8s}")
print("-" * 50)
all_params = list(imps_fanova.keys())
for p in all_params:
    f = imps_fanova.get(p, 0.0)
    m = imps_mdi.get(p, 0.0)
    print(f"{p:25s}  {f:>8.3f}  {m:>8.3f}")

# Spearman rank correlation between the two rankings
fanova_ranks = [list(imps_fanova.keys()).index(p) for p in all_params]
mdi_ranks    = [list(imps_mdi.keys()).index(p)    for p in all_params]
rho, pval = spearmanr(fanova_ranks, mdi_ranks)
print(f"\nSpearman rank correlation: rho = {rho:.3f} (p = {pval:.3f})")

top2_agree = list(imps_fanova.keys())[:2] == list(imps_mdi.keys())[:2]
print(f"Top-2 agree? {'yes' if top2_agree else 'no'}")
print(f"  fANOVA top-2: {list(imps_fanova.keys())[:2]}")
print(f"  MDI top-2:    {list(imps_mdi.keys())[:2]}")

Typical output: fANOVA and MDI agree on the top-2 (learning_rate and max_depth), though possibly in different orders. Spearman rank correlation across all 6 hyperparameters is typically 0.7–0.85 — high but not perfect. The most common disagreement is on n_estimators: MDI tends to rank it higher than fANOVA does, because MDI is biased toward continuous high-cardinality features (and n_estimators ranges over 50–500, giving it high cardinality). fANOVA's decomposition is theoretically better founded for hyperparameter importance; MDI is faster but noisier.

🎯 What This Assignment Teaches

  • Running a 100-trial TPE search is fast (5–10 min on a laptop) and produces enough data for a meaningful analysis.
  • The five standard Optuna visualizations together give a complete picture of what the search found — convergence, joint structure, importances, interactions, marginal effects.
  • For XGBoost on tabular classification, `learning_rate` and `max_depth` typically explain 70%+ of variance, exactly as predicted by the theory in Modules 5–6.
  • The train-vs-val scatter plot is the cheapest overfitting diagnostic — and on a well-behaved dataset, it usually shows no signal.
  • fANOVA and MDI evaluators agree on the top-2 but diverge on the long tail — pick fANOVA for principled rankings, MDI for speed.
