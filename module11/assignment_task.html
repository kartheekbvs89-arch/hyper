Assignment Overview

In this assignment you will run an end-to-end tuning job on a real dataset and produce a complete visualization report. The point is not to "find the best model" — it is to learn how to read a finished study. By the end you should be able to look at the six standard Optuna visualizations and tell, in two minutes, which hyperparameters matter, where the good region lives, and whether your search has converged.

You will tune XGBoost on the load_breast_cancer dataset (569 samples, 30 features, binary target) using Optuna's TPESampler with 100 trials. Six hyperparameters are in scope: learning_rate, max_depth, n_estimators, subsample, colsample_bytree, and min_child_weight. You will generate the five standard visualizations, write a short analysis, and run an overfitting diagnostic. There are two bonus tasks for the ambitious: an fANOVA cross-evaluator comparison and a train-vs-validation analysis.

📌
Required packages

You will need optuna >= 3.0, xgboost >= 1.7, scikit-learn >= 1.2, plotly, and matplotlib. Install with pip install optuna xgboost scikit-learn plotly matplotlib pandas seaborn. Optional for the bonus: mlflow.

Dataset

The load_breast_cancer dataset is a small, well-behaved binary classification problem. It is small enough that 100 trials will finish in a few minutes on a laptop, but large enough that hyperparameter choice has a measurable effect on validation accuracy. The 30 features are real-valued measurements of cell nuclei in breast-mass images; the target is malignant vs benign. Class balance is 357 benign vs 212 malignant — use StratifiedKFold for your cross-validation splitter.
