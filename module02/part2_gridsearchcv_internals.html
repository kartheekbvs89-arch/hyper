1. The Double Loop, Step by Step

When you call grid.fit(X, y), scikit-learn does roughly the following. First, it clones the estimator (a fresh copy with the same hyperparameters but no learned state). Then it enumerates every parameter combination in param_grid — for each combination, it sets the cloned estimator's hyperparameters via set_params. For each (combination, fold) pair, it fits the estimator on the training portion of the fold and scores it on the validation portion. The result is a 2D array of scores: rows are combinations, columns are folds.

The double loop is the source of both the cost (k_cv times more fits than the number of combinations) and the parallelism (every (combination, fold) pair is independent). With n_jobs=-1, joblib dispatches these pairs to worker processes, collects the results, and assembles them into the cv_results_ dictionary.

snippet.py
# ─── gridsearchcv_internals.py ─────────────────────────────────────────────────
# Hand-rolled equivalent of what GridSearchCV does internally, so you can see
# exactly what data structure is being assembled.

import numpy as np
import pandas as pd
from sklearn.base import clone
from sklearn.datasets import load_breast_cancer
from sklearn.svm import SVC
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import make_pipeline
from sklearn.model_selection import StratifiedKFold
from sklearn.metrics import accuracy_score
from itertools import product

X, y = load_breast_cancer(return_X_y=True)

param_grid = {
    "svc__C":     [0.1, 1, 10, 100],
    "svc__gamma": [0.001, 0.01, 0.1, 1],
}
cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)

# ─── Build the list of param dicts (the cartesian product) ───────────────────
keys = list(param_grid.keys())
combos = [dict(zip(keys, vals)) for vals in product(*param_grid.values())]
print(f"{len(combos)} combinations × {cv.get_n_splits()} folds "
      f"= {len(combos) * cv.get_n_splits()} fits")

# ─── The double loop ─────────────────────────────────────────────────────────
scores = np.zeros((len(combos), cv.get_n_splits()))
for ci, params in enumerate(combos):
    for fi, (tr, va) in enumerate(cv.split(X, y)):
        estimator = make_pipeline(StandardScaler(), SVC(kernel="rbf", random_state=42))
        estimator.set_params(**params)
        estimator.fit(X[tr], y[tr])
        scores[ci, fi] = accuracy_score(y[va], estimator.predict(X[va]))

# ─── Assemble a cv_results_-like DataFrame ───────────────────────────────────
results = pd.DataFrame({
    "params": combos,
    "mean_test_score": scores.mean(axis=1),
    "std_test_score": scores.std(axis=1),
    **{f"split{i}_test_score": scores[:, i] for i in range(cv.get_n_splits())},
})
results["rank_test_score"] = results["mean_test_score"].rank(ascending=False).astype(int)
results = results.sort_values("rank_test_score")
print(results[["params", "mean_test_score", "std_test_score", "rank_test_score"]].head())

This is essentially what scikit-learn does internally, modulo parallelism, refit, and some bookkeeping. The point of writing it out by hand is to demystify the cv_results_ dictionary — once you have built one yourself, you will never be confused by its structure again.

2. The cv_results_ Dictionary: Every Key Explained

After fit, grid.cv_results_ is a dict where each value is a list of length n_combinations. The keys fall into a few groups:

KeyTypeWhat it contains
paramslist of dictsThe actual hyperparameter combination for each row.
mean_test_scorearrayCV mean of the validation score — what you usually care about.
std_test_scorearrayStandard deviation across folds — your variance gauge.
rank_test_scorearray of int1 = best mean_test_score; useful for filtering.
split0_test_score … splitK_test_scorearraysPer-fold validation scores — for box plots and stability analysis.
mean_train_score, std_train_score, splitK_train_scorearraysSame for the training fold score (only if return_train_score=True).
mean_fit_time, std_fit_timearraysWall-clock time to fit, averaged across folds.
mean_score_time, std_score_timearraysWall-clock time to score (predict + metric).
param_<name>arraysThe value of hyperparameter <name> for each row — one column per hyperparameter. Used for pivot tables.
param_<name>_fold (some versions)arraysInternal fold indices — usually ignorable.

The most useful keys for analysis are params, mean_test_score, std_test_score, and the per-fold splitK_test_score arrays. With those four, you can rank configurations, compute stability scores, and produce box plots.

3. Ranking by Mean and Consistency

The default rank_test_score ranks by mean_test_score alone. But mean alone is misleading: a configuration with mean 0.92 ± 0.04 is riskier than one with mean 0.91 ± 0.005, because the former might score anywhere from 0.88 to 0.96 on a new sample of the data, while the latter is almost certain to be in [0.905, 0.915]. For deployment you usually want the stable one.

A common alternative ranking is mean − std — the lower end of the 1-sigma confidence band. This penalizes high-variance configurations and rewards consistency. You can compute it from cv_results_ directly.

snippet.py
# ─── rank_by_stability.py ────────────────────────────────────────────────────
# Rank configurations by (mean - std) instead of mean alone.

import pandas as pd
from sklearn.model_selection import GridSearchCV
from sklearn.svm import SVC
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import make_pipeline
from sklearn.datasets import load_breast_cancer
from sklearn.model_selection import StratifiedKFold

X, y = load_breast_cancer(return_X_y=True)
grid = GridSearchCV(
    make_pipeline(StandardScaler(), SVC(kernel="rbf")),
    param_grid={"svc__C": [0.1, 1, 10, 100], "svc__gamma": [0.001, 0.01, 0.1, 1]},
    cv=StratifiedKFold(n_splits=5, shuffle=True, random_state=42),
    scoring="accuracy", n_jobs=-1, return_train_score=True,
)
grid.fit(X, y)

df = pd.DataFrame(grid.cv_results_)
df["mean_minus_std"] = df["mean_test_score"] - df["std_test_score"]
df = df[["params", "mean_test_score", "std_test_score",
         "mean_minus_std", "rank_test_score"]].copy()
df["rank_by_stability"] = df["mean_minus_std"].rank(ascending=False).astype(int)
df = df.sort_values("rank_by_stability")
print(df.head(8).to_string(index=False))

The top configuration by stability is often different from the top by mean — and it is usually the better deployment choice. If you want to use stability ranking for the actual refit step, pass a custom callable to refit as shown in the next section.

4. Custom refit Strategies

The refit parameter accepts either a string (the metric name to refit on, for multi-metric) or a callable. The callable receives the cv_results_ dict and returns the index of the configuration to refit. This lets you implement custom selection logic without forking GridSearchCV.

snippet.py
# ─── custom_refit.py ──────────────────────────────────────────────────────────
# Use a custom refit strategy that picks the configuration with the highest
# (mean - std) — i.e. the most pessimistic lower bound on accuracy.

from sklearn.model_selection import GridSearchCV
from sklearn.svm import SVC
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import make_pipeline
from sklearn.datasets import load_breast_cancer
from sklearn.model_selection import StratifiedKFold
import numpy as np

def refit_lower_bound(cv_results):
    """Pick the index of the configuration with the highest (mean - std)."""
    lower = cv_results["mean_test_score"] - cv_results["std_test_score"]
    return int(np.argmax(lower))

X, y = load_breast_cancer(return_X_y=True)
grid = GridSearchCV(
    make_pipeline(StandardScaler(), SVC(kernel="rbf")),
    param_grid={"svc__C": [0.1, 1, 10, 100], "svc__gamma": [0.001, 0.01, 0.1, 1]},
    cv=StratifiedKFold(n_splits=5, shuffle=True, random_state=42),
    scoring="accuracy", n_jobs=-1, return_train_score=True,
    refit=refit_lower_bound,
)
grid.fit(X, y)
print(f"Refit on config with highest (mean - std): {grid.best_params_}")
print(f"  mean_test_score = {grid.best_score_:.4f}")

Other useful refit strategies: pick the configuration with the lowest fit time among those whose mean_test_score is within 1 std of the best; pick the simplest model (smallest n_estimators, shallowest max_depth) within the top-5 by mean; pick the configuration that maximizes recall subject to precision ≥ 0.9.

5. Memory and Caching: joblib.Memory

If your pipeline contains expensive transformers (PCA on large data, TF-IDF vectorization on text, custom feature engineering), GridSearchCV will refit them on every (combination, fold) pair — even when the transformer's hyperparameters haven't changed. With 16 combinations × 5 folds, that is 80 refits of the transformer, of which perhaps 5 are unique (one per fold). The other 75 are wasted compute.

The fix is joblib.Memory: a disk-based cache that memoizes transformer fits by hashing their inputs and hyperparameters. When the search encounters the same transformer with the same inputs again, it loads the cached result instead of recomputing. The speedup can be 10× or more for text pipelines.

snippet.py
# ─── caching_with_memory.py ──────────────────────────────────────────────────
# Wrap expensive transformers with joblib.Memory so GridSearchCV doesn't
# recompute them on every (combination, fold) pair.

import tempfile
from pathlib import Path
from joblib import Memory
from sklearn.pipeline import Pipeline
from sklearn.decomposition import PCA
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import GridSearchCV
from sklearn.datasets import load_breast_cancer

cache_dir = Path(tempfile.gettempdir()) / "hypertune_cache"
cache_dir.mkdir(exist_ok=True)
memory = Memory(location=str(cache_dir), verbose=0)

X, y = load_breast_cancer(return_X_y=True)

# ─── Attach the memory to the pipeline — every transformer step is cached ───
pipe = Pipeline([
    ("pca", PCA(n_components=10)),
    ("rf", RandomForestClassifier(n_estimators=100, random_state=42)),
], memory=memory)

grid = GridSearchCV(
    pipe,
    # Only RF hyperparameters vary — PCA is recomputed only once per fold
    param_grid={"rf__max_depth": [4, 8, 12, 16]},
    cv=5, n_jobs=-1, scoring="accuracy",
)
grid.fit(X, y)
print(f"Best: {grid.best_params_}, CV acc = {grid.best_score_:.4f}")

Note that caching only helps if the transformer's inputs and hyperparameters are constant across the configurations you are searching over. If you are also tuning pca__n_components, the cache will have one entry per (n_components, fold) pair, which is still useful but less dramatic.

6. Custom Scoring Functions

The scoring parameter accepts strings ('accuracy', 'roc_auc', 'f1', …), but for real problems you often need a business metric. Maybe false positives cost $10 each and false negatives cost $100, and you want to optimize profit-weighted accuracy directly. sklearn.metrics.make_scorer lets you wrap any callable into a scorer that GridSearchCV can use.

snippet.py
# ─── custom_scorer.py ────────────────────────────────────────────────────────
# Define a profit-weighted scorer: false negatives cost 10x false positives.

import numpy as np
from sklearn.metrics import make_scorer, confusion_matrix
from sklearn.model_selection import GridSearchCV
from sklearn.ensemble import RandomForestClassifier
from sklearn.datasets import load_breast_cancer
from sklearn.model_selection import StratifiedKFold

def profit_score(y_true, y_pred):
    """Return a per-sample profit: +1 for correct, -10 for FN, -1 for FP."""
    tn, fp, fn, tp = confusion_matrix(y_true, y_pred, labels=[0, 1]).ravel()
    return (tp + tn - 10 * fn - fp) / len(y_true)

profit_scorer = make_scorer(profit_score, greater_is_better=True)

X, y = load_breast_cancer(return_X_y=True)
grid = GridSearchCV(
    RandomForestClassifier(n_estimators=200, random_state=42, n_jobs=1),
    param_grid={"max_depth": [4, 8, 12, 16], "class_weight": [None, "balanced"]},
    cv=StratifiedKFold(5, shuffle=True, random_state=42),
    scoring=profit_scorer, n_jobs=-1, refit=True,
)
grid.fit(X, y)
print(f"Best by profit: {grid.best_params_}  profit = {grid.best_score_:.4f}")

Notice that class_weight='balanced' will typically win under this scorer even though it might lose under plain accuracy — the asymmetric cost changes which configuration is best. This is exactly why business-aligned scorers matter: they change the answer.

7. The Warm-Start Limitation

Some estimators support warm_start=True, which lets you continue training from where a previous fit left off. This is invaluable for, say, growing a random forest by adding 50 more trees at a time. Unfortunately, GridSearchCV does not use warm start across its own iterations — each (combination, fold) fit is independent, and there is no way to tell the search "remember what you learned from n_estimators=100 when fitting n_estimators=200."

This is a real limitation that more sophisticated tools (Optuna, Hyperopt) handle better. With Optuna you can use a TrailPruner and warm-started estimators to skip redundant compute. The takeaway is: if your search involves incrementally varying a hyperparameter like n_estimators, and warm-starting is supported, consider Optuna instead of GridSearchCV.

📌
Why sklearn doesn't warm-start GridSearchCV

Two reasons. (1) GridSearchCV clones the estimator before each fit, so it cannot inherit learned state across configurations. (2) Warm-starting changes the optimization trajectory — a warm-started fit on n_estimators=200 is not the same model as a fresh fit on n_estimators=200, because the additional 100 trees were trained against residuals computed with the first 100 trees. This breaks the statistical cleanliness of the CV estimate.

8. Putting It All Together: A Production-Quality Search

Let's combine everything from this article into a single production-quality search: caching, custom scorer, stability-based refit, and a thorough cv_results_ analysis. This is the kind of code you should be writing for real tuning jobs, not toy demos.

snippet.py
# ─── production_grid_search.py ─────────────────────────────────────────────────
# Production-quality GridSearchCV: caching, custom scorer, stability refit,
# thorough cv_results_ analysis with stability ranking and box plots.

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
import tempfile
from joblib import Memory
from pathlib import Path
from sklearn.datasets import load_breast_cancer
from sklearn.ensemble import RandomForestClassifier
from sklearn.pipeline import Pipeline
from sklearn.model_selection import GridSearchCV, StratifiedKFold
from sklearn.metrics import make_scorer, confusion_matrix

# ─── Setup ────────────────────────────────────────────────────────────────────
X, y = load_breast_cancer(return_X_y=True)
cache_dir = Path(tempfile.gettempdir()) / "hypertune_prod_cache"
cache_dir.mkdir(exist_ok=True)
memory = Memory(location=str(cache_dir), verbose=0)

# ─── Custom scorer: profit-weighted ──────────────────────────────────────────
def profit_score(y_true, y_pred):
    tn, fp, fn, tp = confusion_matrix(y_true, y_pred, labels=[0, 1]).ravel()
    return (tp + tn - 10 * fn - fp) / len(y_true)
profit_scorer = make_scorer(profit_score, greater_is_better=True)

# ─── Custom refit: highest (mean - std) ──────────────────────────────────────
def refit_stability(cv_results):
    lower = cv_results["mean_test_score"] - cv_results["std_test_score"]
    return int(np.argmax(lower))

# ─── Pipeline + grid ─────────────────────────────────────────────────────────
pipe = Pipeline([
    ("rf", RandomForestClassifier(n_estimators=300, n_jobs=1, random_state=42)),
], memory=memory)

param_grid = {
    "rf__max_depth":        [4, 6, 8, 10, 12, 16],
    "rf__min_samples_leaf": [1, 2, 4, 8],
    "rf__max_features":     ["sqrt", "log2", 0.5],
}

grid = GridSearchCV(
    pipe, param_grid=param_grid,
    cv=StratifiedKFold(n_splits=5, shuffle=True, random_state=42),
    scoring={"profit": profit_scorer, "accuracy": "accuracy"},
    refit=refit_stability,  # refit by stability on the FIRST metric (profit)
    n_jobs=-1, return_train_score=True,
)
grid.fit(X, y)
print(f"Best params: {grid.best_params_}")
print(f"Best profit (mean - std): {grid.best_score_:.4f}")

# ─── Results analysis ────────────────────────────────────────────────────────
df = pd.DataFrame(grid.cv_results_)
df["profit_lower"] = df["mean_test_profit"] - df["std_test_profit"]
top10 = df.nlargest(10, "profit_lower")[
    ["params", "mean_test_profit", "std_test_profit",
     "mean_test_accuracy", "mean_fit_time"]
]
print("\nTop 10 by profit lower bound:")
print(top10.to_string(index=False))

# ─── Box plot of per-fold profit for top 8 configs ──────────────────────────
top8 = df.nlargest(8, "profit_lower")
split_cols = [c for c in df.columns if c.startswith("split") and c.endswith("test_profit")]
fig, ax = plt.subplots(figsize=(10, 5), constrained_layout=True)
ax.boxplot([top8[c].values for c in split_cols], labels=[f"#{i+1}" for i in range(len(top8))])
ax.set_ylabel("Per-fold profit score")
ax.set_xlabel("Top 8 configurations (by mean - std)")
ax.set_title("Stability of top configurations across CV folds")
ax.axhline(grid.best_score_, color="#34d399", linestyle="--", alpha=0.7,
           label=f"Best (mean - std) = {grid.best_score_:.4f}")
ax.legend()
plt.savefig("grid_search_stability.png", dpi=120)
plt.show()

The box plot is the key output. If the top configurations all have tight boxes (small variance across folds), you can be confident in your selection. If the boxes are tall and overlapping, you should run more CV folds, gather more data, or accept that the problem doesn't distinguish strongly between top configs.

🧠 Quick Check

A grid search reports best_score_=0.95 with std=0.012. A colleague reruns the same search with a different random_state on the CV split and gets best_score_=0.91. What is happening, and what should you do?

Show Answer

The two searches picked DIFFERENT best configurations — the second run's 'best' config happened to score 0.91 on the first run's CV splits. This means the configurations are statistically indistinguishable and the 'best' is largely noise. Fix: (1) increase cv to 10 to reduce variance, (2) use refit by (mean - std) to pick a stable config, (3) report the distribution of top-K configs rather than a single best.

🎯 Key Takeaways

  • GridSearchCV runs a double loop: configurations × folds, all independent, parallelizable.
  • cv_results_ contains everything: per-fold scores, timing, params, ranks. Convert to pandas for analysis.
  • Rank by (mean - std) for deployment — consistency matters as much as peak performance.
  • Use joblib.Memory to cache expensive transformers — 10× speedup is routine for text/PCA pipelines.
  • Use make_scorer for business metrics — asymmetric costs change which configuration is best.
  • GridSearchCV cannot warm-start across configurations — use Optuna if you need incremental training.
