Assignment Overview

In production ML, accuracy is never the only objective. You also care about inference latency, memory footprint, training cost, and a dozen other operational constraints. This assignment asks you to tune a model under two simultaneous objectives — validation accuracy and inference latency — using Optuna's NSGAIISampler, then identify two distinct operating points on the resulting Pareto front.

You will set up XGBoost on load_breast_cancer, define a multi-objective study, run 100 trials, plot the Pareto front, and pick operating points under two different constraints. Two bonus tasks extend the work to MLflow logging (so every trial is queryable in the UI) and Ray Tune (so the same study runs across multiple workers).

📌
Required packages

You will need optuna >= 3.0, xgboost >= 1.7, scikit-learn >= 1.2, matplotlib, and numpy. For the bonuses: mlflow >= 2.0 and ray[tune] >= 2.0. Install with pip install optuna xgboost scikit-learn matplotlib numpy mlflow "ray[tune]".

Dataset & Setup

The load_breast_cancer dataset is the same as Assignment 11: 569 samples, 30 features, binary target. Use StratifiedKFold(n_splits=5, shuffle=True, random_state=42) for cross-validation. For latency measurement, use a fixed random subsample of 1000 instances (with replacement if needed) and report the median of 5 repeats of 100 calls to model.predict().
