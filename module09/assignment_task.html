Assignment Overview

In this assignment you will run two AutoML systems on the same tabular benchmark, compare them on both speed and accuracy, and inspect what FLAML actually chose. The goal is not to win a Kaggle competition but to build intuition for when each tool wins — and to see, concretely, the cost-aware escalation strategy FLAML uses under the hood.

The assignment is designed for 3–4 hours of work, including installation time. If you already have FLAML and Optuna installed and the bonus auto-sklearn install goes smoothly, it can be done in 2 hours; budget extra time for the bonus task because auto-sklearn's dependency tree is heavy.

Dataset

We use the German Credit dataset (fetch_openml(name='credit-g', version=1)) — 1,000 rows, 20 mixed-type features (7 numerical, 13 categorical), binary target (good vs bad credit). It is small enough that every AutoML trial finishes in seconds, but rich enough that the algorithm choice matters: a linear model with the right preprocessing gets ~75% accuracy, while a tuned LightGBM gets ~76–77%. The gap is small but real, which makes it a good test of whether your AutoML system finds the better model.

Submission Instructions

Save your notebook as assignment_09_[your_name].ipynb and submit it via your course platform. Make sure every code cell runs from top to bottom in a fresh kernel. If auto-sklearn did not install on your machine, write a 3-sentence note in the Task 4 cell explaining what you tried and submit Tasks 1–3 only — partial credit applies.

💡
Set seeds everywhere

FLAML accepts seed=42; Optuna accepts sampler=TPESampler(seed=42); sklearn train_test_split takes random_state=42; XGBoost takes random_state=42. Set all of them. AutoML results are noisy, and you will be running this assignment multiple times to compare tools — without seeds, the comparison is meaningless.
